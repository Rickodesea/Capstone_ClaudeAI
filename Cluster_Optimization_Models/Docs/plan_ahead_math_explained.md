# Plan-Ahead Model — Mathematical Reference

## 1. Overview

The plan-ahead model is a Mixed-Integer Second-Order Cone Program (MISOCP) solved
periodically (every `plan_ahead_interval` simulation steps).  It answers the question:

> *Which nodes should each tenant be prioritised on for each planning period,
> given their estimated resource demand?*

The output is a **priority hint** — a `TenantAccessSchedule` mapping
`(tenant_id, period) → [node_ids]`.  The real-time scheduler uses this hint
to boost its objective coefficients; it does **not** block any node.

### Two solver modes

The model supports a `use_socp` flag:

| Mode | When used | Capacity constraint |
|------|-----------|---------------------|
| MILP (`use_socp=False`) | Simulation (runs every 50 steps — speed matters) | Plain linear: total allocation ≤ capacity |
| MISOCP (`use_socp=True`) | Standalone plan-ahead (runs once — speed is fine) | Cantelli: allocation + safety buffer ≤ capacity, with cone constraint on buffer size |

The SOCP mode gives a probabilistic capacity guarantee; the MILP mode is identical in structure but drops the safety buffer for faster solving. The simulation defaults to MILP; the standalone model defaults to SOCP.

---

## 2. Sets

| Symbol | Description |
|--------|-------------|
| $\mathcal{T}$ | Tenants, indexed $i \in \{0, \ldots, \lvert\mathcal{T}\rvert - 1\}$ |
| $\mathcal{N}$ | Cluster nodes, indexed $n \in \{0, \ldots, \lvert\mathcal{N}\rvert - 1\}$ |
| $\mathcal{H}$ | Planning periods, indexed $h \in \{0, \ldots, \lvert\mathcal{H}\rvert - 1\}$ |

The number of planning periods is:

$$\lvert\mathcal{H}\rvert = \left\lfloor \frac{\text{plan\_ahead\_interval}}{\text{access\_period}} \right\rfloor$$

---

## 3. Parameters

| Symbol | Description |
|--------|-------------|
| $u_{i,h}$ | Estimated total resource demand of tenant $i$ in period $h$ (capacity units) |
| $C_n$ | Resource capacity of node $n$ (uniform in synthetic data) |
| $\pi_n$ | Infrastructure cost of activating node $n$ per period |
| $\bar{\pi}_i$ | Contract revenue for admitting tenant $i$ |
| $v_i$ | Operational cost for tenant $i$ |
| $\lambda_0, \lambda_1, \lambda_2$ | Objective weights (infra cost, admission revenue, fairness) |
| `sigma_frac` | Demand uncertainty fraction: $\text{std}(D_{i,h}) = \text{sigma\_frac} \times u_{i,h}$ |
| $\sigma^2_{i,h}$ | Per-tenant per-period demand variance: $(\text{sigma\_frac} \times u_{i,h})^2$ |
| $\varepsilon$ | Cantelli tail probability (`cantelli_epsilon`); default 0.10 |
| $\kappa$ | Cantelli safety factor: $\kappa = \sqrt{(1-\varepsilon)/\varepsilon}$; with $\varepsilon=0.10$, $\kappa=3$ |

### Tenant Usage Profiles

$u_{i,h}$ is generated by a **prediction layer** (currently a placeholder using
`np.random.uniform(tenant_usage_min, tenant_usage_max)`).  In production this
comes from historical CollectionEvents SUBMIT counts per tenant per access period.

---

## 4. Decision Variables

| Variable | Domain | Description |
|----------|--------|-------------|
| $f_{i,n,h}$ | $\geq 0$ (continuous) | Allocation of node $n$'s capacity to tenant $i$ in period $h$ |
| $y_{i,n,h}$ | $\{0,1\}$ (binary) | 1 iff tenant $i$ has a priority assignment on node $n$ in period $h$ |
| $z_{n,h}$ | $\{0,1\}$ (binary) | 1 iff node $n$ is active in period $h$ |
| $t_{n,h}$ | $\geq 0$ (continuous) | Cantelli slack — upper bound on weighted demand std dev on node $n$ in period $h$ |
| $a_i$ | $\{0,1\}$ (binary) | 1 iff tenant $i$ is admitted for this planning horizon |
| $\sigma$ | $\in [0,1]$ (continuous) | Minimum demand-satisfaction ratio across admitted tenants (fairness) |

---

## 5. Constraints

### C1a — Capacity with Safety Buffer (linear)

$$\sum_{i \in \mathcal{T}} f_{i,n,h} + \kappa \cdot t_{n,h} \leq C_n \cdot z_{n,h}
\qquad \forall\, n \in \mathcal{N},\; h \in \mathcal{H}$$

The total planned allocation on a node, **plus a safety buffer** $\kappa \cdot t_{n,h}$, must fit within the node's capacity.

The safety buffer exists because $u_{i,h}$ is a prediction — actual usage may be higher.  The buffer grows with uncertainty.  When $\kappa = 0$ (no uncertainty), the buffer disappears and this becomes the plain capacity limit.

### C1b — How Big the Buffer Must Be (cone constraint)

$$\sum_{i \in \mathcal{T}} \sigma^2_{i,h} \cdot y_{i,n,h} \leq t_{n,h}^2
\qquad \forall\, n \in \mathcal{N},\; h \in \mathcal{H}$$

This forces $t_{n,h}$ to be large enough to cover the combined uncertainty of all tenants assigned to that node.

Each tenant's uncertainty is $\sigma^2_{i,h} = (\text{sigma\_frac} \times u_{i,h})^2$ — bigger predicted usage means bigger uncertainty.  The $y_{i,n,h}$ term activates the uncertainty only for tenants actually assigned to node $n$.  This quadratic shape is what makes it a second-order cone constraint.

**What does this achieve?**  C1a and C1b together guarantee that the node will not be overloaded even if actual demand is higher than predicted — with at least $(1 - \varepsilon)$ probability.  With default $\varepsilon = 0.10$, $\kappa = 3.0$: the node stays within capacity at least **90% of the time** even under demand variation.

### C2 — Priority Link

$$f_{i,n,h} \leq C_n \cdot y_{i,n,h}
\qquad \forall\, i \in \mathcal{T},\; n \in \mathcal{N},\; h \in \mathcal{H}$$

If $y_{i,n,h} = 0$, tenant $i$ is assigned no capacity on node $n$ in period $h$.
If $y_{i,n,h} = 1$, up to full node capacity may be allocated.

### C3 — Demand Satisfaction

$$\sum_{n \in \mathcal{N}} f_{i,n,h} \geq u_{i,h} \cdot a_i
\qquad \forall\, i \in \mathcal{T},\; h \in \mathcal{H}$$

Admitted tenants must receive at least their estimated demand each period.

### C4 — Fairness

$$\sigma \leq \frac{\displaystyle\sum_{n \in \mathcal{N},\; h \in \mathcal{H}} f_{i,n,h}}%
{\displaystyle\sum_{h \in \mathcal{H}} u_{i,h}}
+ (1 - a_i)
\qquad \forall\, i \in \mathcal{T}$$

$\sigma$ is the minimum demand-satisfaction ratio across all admitted tenants.
The $(1 - a_i)$ term deactivates this constraint for rejected tenants.
Since $u_{i,h}$ is a parameter (constant), this constraint is **linear**.

### C5 — Node Activation

$$z_{n,h} \geq y_{i,n,h}
\qquad \forall\, i \in \mathcal{T},\; n \in \mathcal{N},\; h \in \mathcal{H}$$

A node must be marked active if any tenant has a priority assignment on it.

---

## 6. Objective

$$\min\quad \lambda_0 \sum_{n,h} \pi_n \cdot z_{n,h}
          - \lambda_1 \sum_i (\bar{\pi}_i - v_i) \cdot a_i
          - \lambda_2 \cdot \sigma$$

The three terms balance infrastructure cost (minimize), admission revenue (maximize),
and fairness (maximize minimum tenant satisfaction).

---

## 7. Output Interface

`extract_tenant_access_schedule(vars_, P)` returns:

```
TenantAccessSchedule = dict[(tenant_id, period) -> list[node_id]]
```

Nodes where $y_{i,n,h} = 1$ are the **priority hints**.  The real-time
scheduler applies a `PRIORITY_BOOST = 2.0` multiplier to the objective
coefficient for `(job, node)` pairs in this set.

All nodes remain accessible regardless — the plan-ahead output is a **soft
priority hint**, not a hard access control.

---

## 8. Connection to Real-Time Model

The real-time solver maximises:

$$\max\; \sum_{j \in J} \sum_{n \in \mathcal{N}}
   \omega_{\text{delay},t(j)} \cdot \hat{m}_j \cdot u_n^{\text{mem}} \cdot \sigma_n^{\text{consolid}}
   \cdot b_{t(j),n} \cdot x_{jn}$$

where

$$b_{t(j),n} = \begin{cases}
  \text{PRIORITY\_BOOST} & \text{if } n \in \text{TenantAccessSchedule}[t(j), h] \\
  1.0                    & \text{otherwise}
\end{cases}$$

Additionally, the job queue is sorted by `sort_by_plan_priority()` before being
passed to the solver, so plan-ahead-prioritised tenants appear first in the
`MAX_JOBS_PER_SOLVE` window.

## 9. Terminology

| Old (original MISOCP) | Current (MISOCP) |
|-----------------------|-----------------|
| "time slot" | "planning period" |
| "workload $j$ of tenant $i$" | "tenant usage profile $u_{i,h}$" |
| "exclusivity constraint" | removed — assignment is priority, not mandate |
| "C5: hard access block" | "PRIORITY\_BOOST objective multiplier" |
| "McCormick linearizations" | not needed (no bilinear terms) |
| "SOCP / Cantelli on workload covariance $\Sigma$" | C1a + C1b: variance-extended Cantelli on $\sigma^2_{i,h} = (\text{sigma\_frac} \times u_{i,h})^2$ |
| "isolation primitives" | removed |
| "migration variables $m_{i,j,n,t}$" | removed |
