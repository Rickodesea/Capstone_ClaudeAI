PLAN-AHEAD MODEL — LEXICON
plan_ahead_lex.txt
================================================================================
One-line definition for every symbol used in the plan-ahead model.
Organised by category. LaTeX form follows the source of truth glossary.
================================================================================


SETS
────────────────────────────────────────────────────────────────────────────────
T            Set of tenants sharing the cluster  {0, 1, ..., |T|-1}
Wi_i         Set of workloads belonging to tenant i  {0, 1, ..., |Wi_i|-1}
N            Set of cluster nodes  {0, 1, ..., |N|-1}
R            Set of resource types  {CPU=0, MEM=1}
H            Set of planning time slots  {0, 1, ..., |H|-1}
Q            Set of QoS classes  {guaranteed=0, burstable=1}
K_prim       Set of isolation primitives  {none=0, gVisor=1, Kata=2}
wl_pairs     All ordered pairs (i,j) < (i',j') of distinct workloads (avoids duplicate co-location vars)
N_allowed_ij Feasible placement nodes for workload (i,j) — compliance/data-residency hard constraint


DECISION VARIABLES
────────────────────────────────────────────────────────────────────────────────
x_ijnt       Binary — 1 if workload j of tenant i is placed on node n at time slot t
y_int        Binary — 1 if tenant i has at least one workload on node n at time t
z_nt         Binary — 1 if node n is active (hosting any workload) at time slot t
w_ijkt       Binary — 1 if isolation primitive k is selected for workload (i,j) at time t
m_mig_ijnt   Binary — 1 if workload (i,j) migrates onto node n at time t (t >= 1 only)
a_i          Binary — 1 if tenant i is admitted in this planning horizon
e_ijt        Continuous >= 0 — excess latency beyond SLA target for workload (i,j) at t
eps_eff_nt   Continuous in [0,1] — effective SLA risk level on node n at t (min of tenants present)
xi_ijnkt     Binary — McCormick auxiliary = x_ijnt * w_ijkt (placement AND primitive)
zeta_...     Binary — McCormick auxiliary = x_ijnt * x_i'j'nt (co-location on same node)
sigma_fair   Continuous in [0,1] — DRF min-satisfaction across admitted tenants (fairness variable)


RESOURCE PARAMETERS
────────────────────────────────────────────────────────────────────────────────
d_ijr        Declared resource demand of workload (i,j) for resource r  [source: InstanceEvents.resource_request]
mu_ijr       Mean predicted resource usage of workload (i,j) for resource r  [source: InstanceUsage.average_usage]
Sigma_r      Empirical covariance matrix of resource r across workloads — PSD, shape (|W_total| x |W_total|)
C_nr         Capacity of node n for resource r  [source: MachineEvents.capacity]
Q_quota_ir   Resource quota for tenant i, resource r (denominator in DRF satisfaction)
alpha_oc_r   Overcommit ratio for resource r (available for soft-overcommit extensions)


PRICING AND RISK PARAMETERS
────────────────────────────────────────────────────────────────────────────────
p_ij         SLA penalty weight for workload (i,j) — maps to scheduling priority in traces
pi_n         Infrastructure cost per active node-slot (operator-defined)
pi_bar_i     Gross revenue from admitting tenant i (billing contract value)
v_op_i       Operational cost of serving tenant i (support overhead)
eps_i        SLA risk tolerance for tenant i — max probability of capacity overrun (Cantelli bound)


ISOLATION PRIMITIVE PARAMETERS
────────────────────────────────────────────────────────────────────────────────
eta_kr       Resource overhead multiplier of isolation primitive k for resource r
             eta_{0,r}=1.0 (none), eta_{1,r}=1.20 (gVisor), eta_{2,r}=1.05 (Kata)
c_k          Operational cost premium for isolation primitive k
rho_kkp      Residual interference between workloads using primitive k and k' on the same node  [0,1]
tau_iip      Maximum tolerable residual interference between tenants i and i'
k_min_i      Minimum required isolation primitive index for tenant i (compliance floor)


CONTROL-PLANE BUDGET PARAMETERS
────────────────────────────────────────────────────────────────────────────────
omega_etcd_ik   etcd write operations consumed per workload of tenant i with primitive k
omega_qps_ik    API-server QPS consumed per workload of tenant i with primitive k
omega_svc_ik    Kubernetes service objects consumed per workload of tenant i with primitive k
B_etcd          Cluster-wide etcd write budget per time slot
B_qps           Cluster-wide API-server QPS budget per time slot
B_svc           Cluster-wide service count budget per time slot
delta_qps       QPS churn cost per migration event (endpoint updates)
B_qps_churn     Maximum allowable migration-induced QPS churn per time slot


SLA AND MIGRATION PARAMETERS
────────────────────────────────────────────────────────────────────────────────
q_assign_ij  QoS class assigned to workload (i,j) — determines latency target
L_iq         Latency SLA target for tenant i, QoS class q  (milliseconds)
gamma_ij     Migration cost weight for workload (i,j) in objective
Delta_i      Per-tenant disruption budget — max migrations allowed per time slot


LATENCY MODEL PARAMETERS
────────────────────────────────────────────────────────────────────────────────
l0_ij            Base latency for workload (i,j) running in isolation  (ms)
alpha_lat_ijijp  Latency increase on (i,j) from co-location with workload (i',j') on the same node
beta_lat_ijk     Latency overhead added by isolation primitive k for workload (i,j)


OBJECTIVE WEIGHTS
────────────────────────────────────────────────────────────────────────────────
lam_0        Weight on infrastructure cost term
lam_1        Weight on SLA penalty term
lam_2        Weight on admission revenue term
lam_3        Weight on fairness (sigma_fair) term
lam_4        Weight on isolation cost term
lam_5        Weight on migration cost term


PROBABILISTIC CAPACITY (SOCP)
────────────────────────────────────────────────────────────────────────────────
kappa_eps    Cantelli one-sided quantile factor: sqrt((1 - eps) / eps)
             kappa(0.05)=4.36 for 95% confidence; kappa(0.10)=3.00 for 90%
kap_global   Conservative global kappa computed at min(eps_i) — applied to all nodes
chol_r       Lower-triangular Cholesky factor of Sigma_r  (chol_r @ chol_r.T = Sigma_r)
soc_aux      Auxiliary variable representing the L2 norm in the SOCP capacity constraint
soc_comp     Auxiliary variable for each row component of chol_r * xi_vec


DERIVED QUANTITIES
────────────────────────────────────────────────────────────────────────────────
MEAN_LOAD_nrt   Mean resource load on node n, resource r, at time t:
                SUM{i,j,k} eta_kr * mu_ijr * xi_ijnkt
dom_r_i         Dominant resource for tenant i:  argmax_r { SUM_j d_ijr / Q_quota_ir }
s_i             DRF satisfaction fraction for tenant i:
                SUM{j,n} d_{i,j,dom_r_i} * x_ijnt / Q_{i,dom_r_i}
net_revenue_i   Net revenue per admitted tenant:  pi_bar_i - v_op_i


CONSTRAINT LABELS
────────────────────────────────────────────────────────────────────────────────
C1       Placement integrity — workload on exactly one node iff tenant admitted (== a[i], not <=)
C1b      Tenant-on-node indicator — y_int=1 whenever any x_ijnt=1
C1c      Compliance/data residency — x_ijnt=0 for nodes not in N_allowed_ij
C2       Probabilistic capacity (SOCP/Cantelli) — mean + kappa*sigma <= C_nr * z_nt
C3       Isolation primitive selection (3 sub-constraints):
           C3a  Exactly one primitive per admitted workload per slot
           C3b  Minimum primitive compliance floor
           C3c  Co-location compatibility (interference tolerance)
C4       Control-plane budget limits (4 sub-constraints):
           C4a  etcd write budget
           C4b  API-server QPS budget
           C4c  Service count budget
           C4d  Migration-induced QPS churn budget
C5       SLA latency satisfaction — linearised latency model with BIG_M deactivation
C6       Migration linking and disruption budget (3 sub-constraints):
           C6a  No migrations at t=0 (no prior state)
           C6b  Migration triggered by relocation
           C6c  Per-tenant disruption budget
C7       DRF fairness — sigma_fair <= s_i for all admitted tenants


OUTPUT
────────────────────────────────────────────────────────────────────────────────
A_t_i              Authorised node set for tenant i at time slot t
                   A_t_i = { n : SUM_j x_ijnt >= 1 }  (derived post-solve)
TenantAccessSchedule   Python dict: (tenant_id, time_slot) -> list[node_id]
                       Passed to real-time model's tenant_node_access parameter


SOLVER SETTINGS
────────────────────────────────────────────────────────────────────────────────
TimeLimit    Maximum solver wall-clock time  (default 300 s)
MIPGap       Relative optimality gap threshold for early termination  (default 1%)
BIG_M        Large constant used in constraint C5 to deactivate for rejected tenants
             Set to max feasible latency value (e.g., 1e4 ms)
