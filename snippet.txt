Kovalenko and Zhdanova (2024) proposed a mathematical model for workload scheduling in cloud environments using a multi-objective function, binary pod-to-node assignment variables, and per-node resource capacity constraints. This project extends that structure with prediction-driven inputs and a fairness objective while removing energy consumption minimization and node power management from scope, as these are outside the boundary of a simulation-based scheduler. Our framework's optimization model has three objectives: maximizing memory utilization across the cluster, maximizing scheduling fairness across tenants, and minimizing SLA violations.

The fairness objective targets per-tenant scheduling delay, defined as the time elapsed between a workload's submission and its assignment to a node (Beltre et al., 2019). In a multi-tenant cluster, tenants whose running workloads hold a disproportionate share of cluster resources leave insufficient capacity on available nodes for other tenants' pending workloads, causing their scheduling delays to grow relative to the cluster average. Beltre et al. (2019) demonstrated this effect empirically: DRF-based scheduling alone produced per-tenant waiting time deviations of up to 157.56% above the cluster average, while combining resource demand-awareness with DRF reduced that deviation to below 13%. This project measures fairness as the variance of per-tenant average scheduling delay relative to the cluster mean. Variance captures inequality directly: when all tenants wait approximately the same amount of time, variance is near zero; when some tenants are systematically delayed, variance is high. Shi and Yu (2024) support this framing theoretically, showing that minimizing queue-length variance across tenants prevents any tenant from experiencing prolonged waiting. The fairness objective in this framework minimizes per-tenant scheduling delay variance, ensuring that utilization gains from overcommitment are not realized at the expense of some tenants over others.

The SLA objective targets uptime. Memory overloading occurs when the total predicted memory consumption assigned to a node exceeds that node's available capacity at a scheduling interval. Each such instance counts as one violation per affected workload per time unit of overloading. This definition follows the principle that memory overcommitment must be bounded to preserve service availability (Kovalenko & Zhdanova, 2024). Unlike CPU throttling, which degrades performance gradually, memory overloading creates immediate risk: a node cannot safely execute workloads beyond its physical capacity without exposing them to termination. One alternative outcome of memory overloading is kernel-level memory swapping, where the operating system moves memory pages to disk to relieve pressure; however, swapping introduces severe I/O latency that degrades all co-located workloads on the node and is disabled by default in Kubernetes clusters (Verma et al., 2015). This project does not model OOM kill recovery, specifically the latency between container termination and successful redeployment, because that requires runtime feedback outside the scope of a scheduling simulation. The SLA objective therefore minimizes cumulative overloading duration across all nodes, which is the measurable uptime proxy a predictive scheduler can directly optimize. In the objective function, this is expressed as the sum of per-node overloading time steps across the simulation window, minimized jointly with the utilization and fairness terms under configurable weighting coefficients.
