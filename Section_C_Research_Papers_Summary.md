| **A Study on Overcommitment in Cloud Providers**<br>Zhi, J. (2025) | AG | Master's dissertation examining overcommitment strategies in cloud providers, focusing on resource allocation beyond physical capacity | Overcommitment (overbooking/multiplexing) allows CSPs to allocate more resources than physically available, improving utilization under the assumption workloads won't peak simultaneously | Provides comprehensive analysis of overcommitment techniques, challenges, and trade-offs in cloud environments | Foundational reference for understanding overcommitment concept, terminology, and implementation challenges in our research context |
| **A Survey of Memory Management Techniques in Virtualized Systems**<br>Mishra, D., Kulkarni, P. (2018) | AG | Comprehensive survey of memory virtualization challenges and management techniques across multiple guest OSs in virtualized environments | Traditional memory sharing via segmentation/paging faces new challenges in virtualization due to dual control (guest OS + hypervisor) and lack of direct hardware access | Details memory partitioning challenges, lack of semantics regarding VM memory usage, and optimization techniques to increase memory utilization | Essential for understanding technical constraints and available techniques for memory management in virtualized multi-tenant environments |
| **Dominant Resource Fairness (DRF)**<br>Ghodsi, A., Zaharia, M., et al. (2011) | AG | Proposes DRF as generalization of max-min fairness to multiple resource types, implemented in Mesos cluster resource manager | Fair resource allocation in systems with heterogeneous demands requires considering each user's dominant resource (resource with maximum share allocated) | DRF satisfies sharing incentive, strategy-proofness, envy-freeness, and Pareto efficiency; achieves better throughput and fairness than slot-based fair sharing | Provides mathematically rigorous framework for ensuring fairness among tenants with different resource demands, directly applicable to our fairness objective |
| **Memory Resource Management in VMware ESX Server**<br>Waldspurger, C.A. (2002) | AG | Introduces novel mechanisms for managing memory in VMware ESX Server to support workloads that overcommit memory | Combines ballooning (reclaiming pages guest OS considers least valuable), idle memory tax, content-based page sharing, and hot I/O page remapping | Techniques efficiently support memory overcommitment while maintaining performance isolation; ballooning allows guest OS to determine which pages to reclaim | Seminal work demonstrating practical implementation of memory overcommitment; ballooning technique particularly relevant for maintaining performance while overcommitting |
| **Resource Central: Understanding and Predicting Workloads**<br>Cortez, E., Bonde, A., et al. (2017) | AG | Microsoft Azure workload characterization study showing VM behaviors are consistent over multiple lifetimes, enabling prediction-based resource management | History is accurate predictor of future VM behavior; machine learning can improve resource management in large cloud platforms | VM telemetry collection and offline learning enables online predictions for various resource managers; prediction-informed scheduling increases utilization while preventing resource exhaustion | Demonstrates value of workload prediction for oversubscription decisions; validates using historical data to optimize resource allocation |
| **Maximizing Server Utilization while Meeting Critical SLAs**<br>Blagodurov, S., Gmach, D., et al. | AG | Presents weight-based collocation management for critical (user-interactive) and non-critical (batch) workloads on virtualized multicore servers | Work-conserving approach with Linux cgroups weights prioritizes critical workload resource access, unlike cap-based solutions that lead to underutilization | Achieved nearly 100% server utilization while keeping critical workload performance loss within specified limits | Directly addresses our core challenge: maximizing utilization while meeting SLAs through priority-based resource allocation |
| **Virtual Machine Memory Balancing (VMMB)**<br>Min, C., Kim, I., et al. (2012) | AG | Novel mechanism to dynamically monitor memory demand and periodically rebalance memory among VMs without modifying guest OS | Accurate measurement of memory demand with low overhead enables effective allocation based on demand and QoS requirements | Improved performance of memory-starved VMs by up to 3.6x with <1% monitoring overhead; avoids double paging problem | Demonstrates practical dynamic memory rebalancing approach applicable to real-time adjustment of memory allocations in our system |
| **Coach: Exploiting Temporal Patterns for All-Resource Oversubscription**<br>Reidys, B., Zardoshti, P., et al. | AG | System exploiting temporal patterns in resource usage to enable safe oversubscription of all resource types in cloud platforms | Temporal patterns in workload behavior can be leveraged to predict when resources can be safely oversubscribed across multiple resource dimensions | Successfully implements all-resource oversubscription by learning and exploiting temporal usage patterns | Extends overcommitment beyond single resources to multi-resource scenarios, aligning with our multi-objective optimization approach |
| **Intelligent Resource Allocation via Machine Learning**<br>Wang, Y., Yang, X. (2025) | AG | Proposes intelligent allocation using LSTM for demand prediction and DQN (reinforcement learning) for dynamic scheduling | ML-based prediction and real-time adjustment can significantly improve resource utilization while maintaining service quality | Enhanced utilization by 32.5%, reduced response time by 43.3%, lowered costs by 26.6% in production cloud environment | Validates ML approaches for demand forecasting and dynamic scheduling, potentially applicable to our prescriptive analytics approach |
| **Mitigating Application Resource Overload**<br>Hu, Y., Zhang, Z., et al. (2025) | AG | Framework (Atropos) for overload control that proactively cancels culprit requests causing resource contention rather than victims | Traditional global signals (queuing delays) fail for application resource overload; targeted cancellation of resource-monopolizing requests is more effective | Maintains performance goals with minimal request drops, outperforming state-of-the-art in 16 real-world overload scenarios | Provides strategy for handling peak demand scenarios when overcommitment leads to contention, complementing our prevention-focused approach |
| **Optimizing Server and Memory Utilization**<br>Krishnaiah, V.V.J.R., Rao, B.S. (2025) | AG | System leveraging virtualization and cache memory to allocate memory based on end-user requirements and address server underutilization | Memory allocation to VMs based on dynamic requirements using cache optimization can reduce underutilization | Addresses challenge of maximum server utilization with increasing end-users; discusses memory overload impacts on application performance | Provides context on server underutilization problem and cache-based optimization techniques |

**Key Research Themes Summary:**

1. **Memory Overcommitment Techniques:**
   - Ballooning, page sharing (Waldspurger, 2002)
   - Dynamic balancing (Min et al., 2012)
   - Temporal pattern exploitation (Reidys et al.)

2. **SLA Guarantees & Performance Isolation:**
   - Weight-based prioritization (Blagodurov et al.)
   - Performance isolation mechanisms (Waldspurger, 2002)
   - Targeted overload mitigation (Hu et al., 2025)

3. **Fairness Frameworks:**
   - Dominant Resource Fairness (Ghodsi et al., 2011)
   - Strategy-proof allocation mechanisms
   - Envy-free resource distribution

4. **Workload Prediction & Characterization:**
   - Historical behavior patterns (Cortez et al., 2017)
   - ML-based demand prediction (Wang & Yang, 2025)
   - Real-time monitoring (Doukha & Ez-zahout, 2025)

5. **Multi-Objective Optimization:**
   - Utilization vs. SLA trade-offs
   - Fairness vs. efficiency balance
   - Cost optimization with quality constraints
